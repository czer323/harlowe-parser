### Error Context Integration Plan

1. **Leverage Chevrotain Token Location Info**
   - In your custom error provider (e.g., `src/errorProvider.ts`), access the `startLine`, `startColumn`, `endLine`, and `endColumn` properties from Chevrotain tokens.
   - When an error is detected, extract these properties from the offending token and include them in the error object.

2. **Extract Relevant Input Snippet**
   - In your parse API (e.g., `src/index.ts`), keep a reference to the original input string.
   - When reporting an error, use the token’s `startOffset` and `endOffset` to extract a substring from the input (e.g., 10 characters before and after the error location).
   - Include this snippet in the error object for context.

3. **Add Actionable Suggestions**
   - In your custom error provider, analyze the error type and context.
   - For common mistakes (e.g., missing closing brackets, unexpected tokens), append a suggestion to the error message (e.g., “Did you forget a closing bracket?”).
   - Maintain a mapping of error types to suggestions for consistency.

4. **Structure Error Objects**
   - Define a TypeScript interface for error objects, e.g.:

     ```ts
     interface ParseError {
       message: string;
       line: number;
       column: number;
       snippet: string;
       suggestion?: string;
       type: string;
     }
     ```

   - Ensure all errors returned by the parse API conform to this structure.

5. **Test Error Context**
   - Write Jest tests that intentionally trigger errors.
   - Assert that the error object includes correct line/column, a relevant input snippet, and a helpful suggestion.

6. **Document Error Context**
   - In the README and API docs, provide examples of error objects and explain how to interpret the context and suggestions.
